# Directory Structure

## Overview
The application is structured around Next.js App Router conventions and clearly separates UI from business logic.

## Key Directories
- `app/`: Next.js application routes.
  - `admin/`: Admin panels.
  - `daily-attendance/`: The primary attendance tracking UI.
  - `login/` & `register/`: Authentication pages.
  - `onboarding/`: User setup flow.
  - `calendar-dashboard/`: Visual dashboards.
  - `semester-configuration/` & `semester-statistics/`: Academic semester management.
  - `simulate/`: Data simulation tools.
  
- `components/`: UI layer.
  - `admin/`: Admin-specific elements.
  - `common/`: Shared layouts/pieces.
  - `ui/`: Core shadcn components.
  
- `lib/`: Business logic, utilities, configuration.
  - `admin/`, `attendance/`, `engines/`, `notes/`: Domain-specific utility functions.
  - `supabase/`: Database client configurations.
  - `utils/`: Generic helpers (`utils.ts`).

- `server/`: Server actions and API routes.
  - `auth/`: Authentication server actions.

- `db/`: Database related files.
  - `seed/`: Local dev seeding scripts.
  - `supabase-schema.sql`: Source of truth for database schema.

- `types/`: Custom global types.
  - `supabase.ts`: Autogenerated Supabase DB typings.
